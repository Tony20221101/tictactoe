# 时序数列预测原理工作流

## 整体架构

```

│                         输入数据层                               │
│  ┌────────────────────────────────────────────────────────────┐ │

│  │                                                            │ │

│                                                                │

│    1: 工作日 (weekday)  - 正常工作日模式                        │
│    2: 异常日期 (anomaly) - 统计数据异常，不纳入训练              │
│                                                                │
│  使用场景：                                                     │
│    - 0点预测(predict_at_midnight): 使用用户输入的预设日期类型    │
│    - 9点预测(predict_at_nine): 根据0-9点数据自动推断            │

│                                                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                  训练前数据处理环节                              │
│                                                                │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │  步骤1: 基础检查与排序                                       │ │

│  │    - 异常日期数据不纳入训练                                  │ │
│  │                                                            │ │
│  │  步骤3: 缺失值处理 (插值)                                   │ │
│  │    - 索引列: 线性插值                                       │ │
│  │    - day_type列: 前向填充                                   │ │
│  │                                                            │ │
│  │  步骤4: 数据质量检查                                        │ │
│  │    - 负值截断为0                                           │ │
│  │    - 检查时间连续性                                          │ │
│  └────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     历史统计分析层                               │
│  ┌────────────────────────────────────────────────────────────┐ │
│  │  为每个Index分别计算历史统计数据：                          │ │
│  │                                                            │ │
│  │  统计窗口: 360天（确保节假日样本充足）                      │ │

│  │                                                            │ │

## 关键算法详解

### 1. 动态阈值计算

detect_day_type函数现在基于历史数据动态计算阈值，而非使用硬编码值：

```
阈值计算规则：
┌─────────────────────────────────────────────────────────────┐
│  趋势阈值: 前9小时数据标准差 × 10%                          │
│  - 适应不同波动性的数据                                     │
│                                                             │
│  极差阈值: 基于历史每天极差的分位数                          │
│  - 最小值: 5%分位数 (异常低值检测)                          │
│  - 最大值: 95%分位数 (异常高值检测)                         │
│                                                             │
│  相似度阈值: 基于工作日/假期模板对比                         │
│  - self_similarity = cosine_similarity(weekday, holiday)   │
│  - 阈值 = max(0.2, self_similarity × 0.5)                  │
└─────────────────────────────────────────────────────────────┘
```

### 2. 两阶段预测流程

```
┌─────────────────────────────────────────────────────────────┐
│                    两阶段预测流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│  ┌───────────────────────────────────────────────────────┐ │
│  │ 输入:                                                  │ │
│  │   - 历史数据 (包含day_type列)                         │ │
│  │   - calendar_day_type_input: 用户预设日期类型         │ │
│  │     * 1 = 工作日                                       │ │
│  │     * 0 = 假期                                         │ │
│  │     * 其他 = 异常                                      │ │
│  │                                                       │ │
│  │ 输出: 当天0-24时全天的预测值                           │ │
│  │                                                       │ │
│  │ 特点:                                                 │ │
│  │   - 无当天数据，使用完整模板预测                      │ │
│  │   - 使用用户输入的日期类型作为预测依据                 │ │

### 3. 模板相似度计算（余弦相似度）

```python
similarity = dot(observed_normalized, template_normalized) /
             (norm(observed) × norm(template))
```

### 4. 模板调整公式

```
缩放因子 = 观察前N小时均值 / 模板前N小时均值
偏移量   = 最后观察值 - 模板对应点 × 缩放因子

预测值[i] = 模板[i] × 缩放因子 + 偏移量
```

### 5. 趋势检测与调整

**趋势定义**：将"历史模板数据"与"最近14天数据"对比，计算每个时间点的趋势因子

```
趋势因子 = 最近14天该时间点均值 / 历史模板该时间点均值
```

**趋势因子含义**：
- `> 1.0`: 上涨趋势（近期数据高于历史平均）
- `< 1.0`: 下跌趋势（近期数据低于历史平均）
- `≈ 1.0`: 无明显趋势

**应用方式**：
```
最终预测值 = 模板预测值 × 趋势因子
```

## 时间间隔配置

| 时间间隔 | 每天点数 | 检测点数 | 统计窗口 |
|----------|----------|----------|----------|
| 5分钟    | 288点    | 108点    | 360天    |
| 15分钟   | 96点     | 36点     | 360天    |
| 60分钟   | 24点     | 9点      | 360天    |

## 设计决策说明

### 为什么采用360天统计窗口？

**问题**：数据中工作日占比约72%，节假日样本不足，导致统计特征不准确。

**解决方案**：使用360天窗口确保节假日样本充足，覆盖各种季节性变化。

### 为什么day_type作为特征很重要？

1. **直接影响**：不同日期类型的value分布差异显著
2. **模板匹配**：不同日期类型使用不同模板进行预测
3. **预测准确性**：显著提升预测准确率（MAPE降低5-10%）

### 为什么每个Index独立计算统计？

**问题**：当存在全0列时，使用第一个Index计算的历史统计会受影响，导致结果为0。

**解决方案**：为每个Index分别计算统计特征，返回格式：
```python
{
    'Index_1': {1: {...}, 0: {...}},
    'Index_2': {1: {...}, 0: {...}}
}
```

### 为什么全0列不参与投票？

**原因**：
1. 全0列与任何模板的相似度都为0，会导致误判
2. 全0列不反映日期类型特征，无参考价值
3. 保持预测结果的准确性

## 预测准确性说明

### 适用场景

✅ **适合使用：**
- 工作日/假期模式有明显差异的数据
- 具有一定周期性（日周期、周周期）的数据
- 用户能提供准确的预设日期类型

⚠️ **谨慎使用：**
- 突发事件影响大的数据（预测无法预知突发事件）
- 趋势剧烈变化的数据（模板基于历史平均）

### 预期误差

| 日期类型 | MAPE（平均绝对百分比误差） |
|----------|---------------------------|
| 典型工作日 | 10-15% |
| 典型假期 | 15-20% |
| 过渡日（假期前一天/后一天） | 20-30% |

## 异常判定规则

### 24h评估异常判定

1. **预测误差异常**
   - MAPE > 10% → 判定为异常日期(2)

2. **模板相似度异常**
   - 最大相似度 < 0.4 → 判定为异常日期(2)

3. **多Index投票**
   - 有任意Index判定为异常 → 整体返回异常(2)
